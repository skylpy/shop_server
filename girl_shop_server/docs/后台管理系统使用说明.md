# Girl Shop 后台管理系统使用说明

## 1. 项目简介

本项目已从原始的静态 Mock 数据服务升级为一个适合演示和二次开发的商城后台项目，核心能力如下：

- 基于 `Node.js + Express + MongoDB`
- 提供独立后台管理系统页面
- 支持商品、用户、分类、订单、轮播图管理
- 保留原有 Flutter 商城接口结构
- 后台修改数据后，商城接口同步读取 MongoDB 中的最新数据

后台入口地址：

- `http://localhost:3000/admin/`

默认管理员账号：

- 账号：`admin`
- 密码：`Admin@123456`

默认商城测试账号：

- 手机号：`13800000001`
- 密码：`User@123456`

## 2. 技术架构

### 服务端

- `Express`：提供后台 API 与商城 API
- `Mongoose`：负责连接 MongoDB
- `MongoDB`：作为项目主数据库

### 前端后台

- 原生 `HTML + CSS + JavaScript`
- 无额外前端框架，方便直接接入当前项目

### 数据集合

项目启动后会自动初始化以下集合：

- `settings`
- `admin_users`
- `categories`
- `products`
- `home_tabs`
- `home_sections`
- `banners`
- `floor_sections`
- `users`
- `orders`
- `operation_logs`

图片上传目录：

- 后台上传的图片会保存到 `public/images/uploadImage/`

## 3. 快速启动

### 方式一：使用本地 MongoDB

1. 安装并启动本地 MongoDB 服务
2. 配置环境变量：

```bash
export PORT=3000
export MONGODB_URI=mongodb://127.0.0.1:27017/girl_shop_admin
export JWT_SECRET=please_change_this_jwt_secret
```

3. 安装依赖并启动项目：

```bash
npm install
npm start
```

### 方式二：使用 Docker 启动 MongoDB

项目已提供 `docker-compose.yml`。

1. 启动 MongoDB：

```bash
docker compose up -d
```

2. 启动后端服务：

```bash
export PORT=3000
export MONGODB_URI=mongodb://127.0.0.1:27017/girl_shop_admin
export JWT_SECRET=please_change_this_jwt_secret
npm start
```

3. 可选查看 MongoDB Web 管理界面：

- `http://localhost:8081`

## 4.1 JWT 接入说明

本项目当前已完成 JWT 接入，后台管理端与商城用户端统一使用 Bearer Token 鉴权。

JWT 相关实现位置：

- `lib/security.js`：负责 JWT 签发、校验、请求头 token 解析
- `router/admin.js`：后台管理员登录与鉴权
- `router/user.js`：商城用户注册、登录与鉴权

JWT 配置说明：

- 环境变量名：`JWT_SECRET`
- 未显式配置时，项目会使用内置演示密钥
- 生产环境务必修改为你自己的高强度随机字符串

JWT 载荷说明：

- 后台 token 会写入 `tokenType=admin`、`adminId`、`username`、`role`
- 商城 token 会写入 `tokenType=user`、`userId`、`phone`
- token 过期时间与返回字段 `expiresAt` 一致，默认读取 `settings.sessionHours`

鉴权方式：

- 请求头统一使用 `Authorization: Bearer <token>`
- 后台接口主要保护 `/api/admin/*`
- 商城接口主要保护 `/api/user/profile` 与 `/api/user/logout`

退出说明：

- 当前 `logout` 接口为前端退出辅助接口
- 服务端不保存 JWT 会话，因此退出登录的关键动作是前端删除本地 token
- 如需让 token 立即失效，可继续扩展 JWT 黑名单或 Redis Session

## 5. 后台功能说明

### 5.1 运营概览

进入后台后默认显示运营概览页面，提供：

- 累计成交额
- 订单总数
- 在售商品数量
- 启用用户数量
- 最近订单列表
- 热销商品排行

### 5.2 商品管理

支持以下操作：

- 按商品名称、商品 ID 搜索
- 按商品状态筛选
- 按分类筛选
- 新增商品
- 编辑商品
- 上架 / 下架商品
- 删除商品

商品字段包括：

- 商品名称
- 副标题
- 一级分类 / 二级分类
- 现价 / 原价
- 库存 / 销量
- 封面图
- 图库
- 标签
- 商品描述
- 热卖状态
- 首页推荐状态

补充说明：

- 商品封面图和图库支持在后台表单中直接上传图片
- 上传成功后会自动回填图片路径到对应输入框

### 5.3 用户管理

支持以下操作：

- 按昵称、手机号、邮箱搜索
- 按状态筛选
- 新增用户
- 编辑用户资料
- 启用 / 禁用用户

用户字段包括：

- 昵称
- 手机号
- 邮箱
- 会员等级
- 城市
- 累计消费金额
- 订单数
- 状态

补充说明：

- 商城登录密码以 `passwordHash` 形式保存在 `users` 集合中，仅用于商城鉴权，后台列表不会回显该字段

### 5.4 分类管理

支持以下操作：

- 新增一级分类
- 编辑分类名称和分类图片
- 配置二级分类
- 删除未被商品使用的分类

说明：

- 如果某个分类下仍有关联商品，系统会阻止删除，避免业务数据失效。
- 分类图片支持直接上传，上传后会自动回填路径

### 5.5 订单管理

支持以下操作：

- 按订单号或用户名搜索
- 按订单状态筛选
- 更新订单状态

支持的订单状态：

- `pending`
- `paid`
- `shipped`
- `completed`
- `cancelled`

### 5.6 轮播管理

支持以下操作：

- 新增轮播图
- 编辑轮播图
- 删除轮播图
- 调整排序
- 切换发布状态

轮播图会直接影响商城接口：

- `/getTestData`
- `/getHomePageContent`

补充说明：

- 轮播图片支持直接上传，上传后会自动回填路径

### 5.7 操作日志与审计记录

项目已增加操作日志与审计记录能力，主要用于后台运维排查、安全审计和问题追踪。

后台页面入口：

- 登录后台后，可在左侧导航点击“操作日志”进入可视化日志页面
- 页面支持按关键字、模块、结果、身份类型和返回条数进行筛选
- 每条日志都可以点击“详情”查看完整信息
- 详情中会展示请求详情和返回详情数据，便于排查问题

当前会自动记录的内容包括：

- 后台管理员登录、退出登录
- 商城用户注册、登录、退出登录
- 后台商品新增、编辑、状态变更、删除
- 后台用户新增、编辑、状态变更
- 后台分类新增、编辑、删除
- 后台订单状态更新
- 后台轮播图新增、编辑、删除

日志记录字段包括：

- 操作模块 `module`
- 操作动作 `action`
- 操作结果 `result`
- 操作人类型、ID、名称
- 目标对象类型、ID、名称
- 请求方法、请求路径、IP、User-Agent
- 返回详情数据 `responseData`
- 状态码、结果说明、创建时间

安全说明：

- 日志中的 `token`、`passwordHash` 等敏感字段会自动脱敏后再保存

查看接口：

- `GET /api/admin/operation-logs`

图片上传接口：

- `POST /api/admin/upload-image`

上传说明：

- 只允许上传图片文件
- 单张图片大小不超过 `10MB`
- 上传成功后返回相对路径和完整访问地址
- 上传文件会保存到 `public/images/uploadImage/`

常用筛选参数：

- `keyword`
- `module`
- `action`
- `result`
- `actorType`
- `limit`

## 6. 后台操作手册

### 6.1 登录后台

1. 启动 MongoDB 和后端服务
2. 浏览器打开 `http://localhost:3000/admin/`
3. 输入管理员账号密码
4. 登录成功后进入后台首页

### 6.2 新增商品

1. 进入“商品管理”
2. 点击“新增商品”
3. 填写商品信息
4. 设置商品分类、价格、库存、封面图、图库
5. 选择是否设为热卖和首页推荐
6. 点击“创建商品”

### 6.3 编辑商品

1. 在商品列表中找到目标商品
2. 点击“编辑”
3. 修改内容后点击“保存修改”

### 6.4 上下架商品

1. 在商品列表中找到目标商品
2. 点击“上架”或“下架”
3. 商品状态更新后，前台接口会同步变化

### 6.5 新增用户

1. 进入“用户管理”
2. 点击“新增用户”
3. 填写用户昵称、手机号、邮箱、等级等信息
4. 点击“创建用户”

### 6.6 维护分类

1. 进入“分类管理”
2. 点击“新增分类”或“编辑”
3. 二级分类可使用逗号输入多个名称
4. 保存后立即生效

### 6.7 更新订单状态

1. 进入“订单管理”
2. 找到目标订单
3. 在订单状态下拉框中选择目标状态
4. 点击“更新状态”

### 6.8 维护首页轮播

1. 进入“轮播管理”
2. 点击“新增轮播”或“编辑”
3. 选择关联商品
4. 设置图片地址、排序、状态
5. 保存后前台首页轮播会同步更新

商城用户鉴权说明：

- 登录和注册成功后会返回 `token`
- 访问 `GET /api/user/profile`、`POST /api/user/logout` 时，请在请求头中携带 `Authorization: Bearer <token>`
- 后台新增但未设置密码的用户，可通过注册接口使用同一手机号完成账号激活
- 当前项目已接入 JWT，默认有效期读取 `settings.sessionHours`

## 8. 注意事项

- 当前项目的后台会在首次启动时自动向 MongoDB 写入初始化数据
- 如果你清空数据库，重启服务后会重新写入种子数据
- 当前后台与商城登录态都使用 JWT，服务重启后不会使已签发 token 立即失效
- 如果修改了 `JWT_SECRET`，旧 token 会全部失效，客户端需要重新登录
- 操作日志和审计记录保存在 `operation_logs` 集合中，`resetDatabase` 后会一并清空
- 适合演示、中小型项目起步、二次开发，不建议直接作为生产环境最终版本上线
